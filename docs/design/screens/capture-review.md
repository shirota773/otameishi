# キャプチャレビュー画面

Version 1.0

---

## 1. 目的

撮影・補正済みの名刺画像と OCR 結果を確認・修正し、タグ・イベントを付与して保存する。

---

## 2. エントリポイント

- キャプチャ画面のシャッター → 処理完了後

---

### レイアウト仕様

- AppBar: `color.surface.primary` 背景、左に `close` アイコン (44×44)
- 名刺画像: 水平 `space.4` マージン、比率固定
- フォームフィールド: `color.surface.tertiary` 背景、`radius.sm`, 高さ 52dp
- 信頼度低フィールド: 右端に `!` アイコン (`color.semantic.warning`)、フィールド枠 `color.semantic.warning`
- sticky footer: `color.surface.primary` 背景、上ボーダー `color.surface.tertiary` 1dp
- キーボード表示時: コンテンツをキーボード上にスクロール (resizeToAvoidBottomInset: true)

## 4. 状態

### 4-1. OCR処理中 (loading)

- 名刺画像プレースホルダー: シマー
- 各テキストフィールド: スケルトン (幅 70% のグレーバー)
- 「保存する」ボタン: 非活性

### 4-2. OCR完了 (populated)

上記ワイヤーフレームの状態

### 4-3. OCR結果ゼロ

- すべてのフィールドが空白
- 警告バナー: `review.ocr.warning`
- 「再読み取り」ボタン表示 (type.label, color.brand.primary)
- ユーザーは手動入力して保存可能

### 4-4. 保存中

- 「保存する」ボタン内にスピナー表示
- すべての入力フィールドを無効化 (フォーカス不可)

### 4-5. 保存完了

- スナックバー: `"保存しました！"` (3秒表示)
- スナックバー内アクション: 「もう1枚追加」 → キャプチャ画面に戻る
- 画面遷移: カード詳細画面へ (`replace`)

### 4-6. 保存エラー

- スナックバー: `review.error.save` + 「再試行」アクション
- フォームに留まる (入力内容は保持)

---

## 5. アクション

| アクション | コンポーネント | 遷移先 |
|---|---|---|
| 保存 (主) | 「保存する」ボタン (52dp) | → 保存中 → カード詳細 |
| 破棄 | 「破棄する」テキストボタン | 確認ダイアログ → ホーム |
| 戻る | AppBar `close` アイコン | 確認ダイアログ → キャプチャ画面 |
| タグ追加 | 「+ 追加」タグチップ | タグ選択ボトムシート |
| イベント選択 | イベントドロップダウン | イベント選択ボトムシート |
| 再読み取り | 「再読み取り」テキストボタン | OCR再実行 → フィールド更新 |

## 7. ダーク/ライトモード差分

| 要素 | ライト | ダーク |
|---|---|---|
| 画面背景 | `color.surface.primary` (#FFFFFF) | `color.surface.primary` (#121212) |
| フィールド背景 | `color.surface.tertiary` (#EEEEEE) | `color.surface.tertiary` (#2A2A2A) |
| 名刺画像シャドウ | `elevation.low` ライト値 | `elevation.low` ダーク値 |

---

## 8. エッジケース

| 状況 | 対処 |
|---|---|
| 画像が縦向き名刺 | 比率を縦 (55:91) に自動切替 (backend が縦横判定) |
| X ID に @ が含まれない | `@` を自動付与して表示 |
| URL が複数抽出 | 最初の1件のみ表示。残りは「他 N件」ラベル + タップで全件表示 |
| 表示名が40文字超 | フィールド内でスクロール可 (1行固定しない) |
| イベントが未作成 | 「イベントなし」をデフォルト選択、「新しいイベントを作成」オプションあり |
