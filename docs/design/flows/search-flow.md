# 検索フロー

Version 1.0

---

## フロー概要

FTS5 ベースのオフライン全文検索。名前・X ID・タグ・メモ・イベント名を横断検索する。

---

## 全体ステップ図

```
[ボトムナビ「検索」タップ]
        │
        ▼
[検索画面 (初期状態)]
  検索バーにフォーカス (キーボード自動表示)
  「最近の検索」リスト表示
        │
        │ 文字入力 (最低1文字)
        ▼
[インクリメンタル検索結果]  ← 入力の都度 SQLite FTS5 クエリ
  デバウンス: 300ms
        │
        ├──[0件] ── 空状態表示
        │
        ▼
[検索結果一覧]
  カード一覧と同じカードサムネイルコンポーネントを再利用
        │
        タップ
        ▼
[カード詳細画面]
  バックで検索結果に戻る
```

---

## 検索対象フィールド

| フィールド | 検索方法 | 備考 |
|---|---|---|
| 表示名 | FTS5 部分一致 | |
| X ID | FTS5 部分一致 | @ 不要でも一致 |
| タグ名 | FTS5 完全一致優先 | スペース区切り複数タグ対応 |
| メモ | FTS5 部分一致 | |
| イベント名 | FTS5 部分一致 | |
| URL | FTS5 部分一致 | ドメイン名も対象 |

---

## 各ステップ詳細

### 初期状態

- 検索バーにオートフォーカス (iOS: `autofocus: true`)
- 「最近の検索」: 直近10件を新しい順で表示 (端末ローカル保存)
- 各最近の検索項目: タップで再検索、右端の × で個別削除
- 「履歴を消去」リンク: 全件削除 (確認ダイアログなし、即時)

### 入力中状態

- デバウンス 300ms 後にクエリ実行
- ローディングインジケーター: 検索バー右端に小スピナー (結果カード入れ替えなし)
- ハイライト: 検索キーワードに一致するテキスト部分を `color.brand.primary` で太字ハイライト

### 結果表示

- カードサムネイルグリッド (2カラム) + マッチしたフィールド名を `type.caption` で小表示
  例: 「タグ: Vtuber」「メモ内で一致」
- 件数表示: `search.result.count` ({count}件) をグリッド上部に表示
- 結果ソート: 関連度スコア降順 (FTS5 bm25)

### 空状態 (0件)

- `search.empty.result.headline` + `search.empty.result.body`
- イラスト: 虫眼鏡アイコン (large, `color.text.tertiary`)

## 特殊ケース

| ケース | 対処 |
|---|---|
| 入力1文字 | 検索実行 (最短1文字から) |
| スペースのみ | クエリ送信しない、初期状態を維持 |
| 絵文字入力 | サポート対象外。無結果として扱う |
| 50文字超 | 入力をトランケート (50文字上限) |
| オフライン | 完全オフライン動作。ネットワーク不要 |
| 5,000件以上のデータ | クエリ ≤ 1秒の性能要件を維持 (FTS5 インデックス必須) |
| カード削除直後の検索 | 削除済みカードは即時除外 (SQLite トリガーでFTSテーブル同期) |
