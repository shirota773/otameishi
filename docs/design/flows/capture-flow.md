# キャプチャフロー — 撮影 → OCR確認 → 保存

Version 1.0

---

## フロー概要

コールドランチから名刺保存完了まで、ユーザーが踏むステップ。

**非機能要件**: コールドランチ → カメラ起動 ≤ 3タップ (理想は1タップ)

---

## 全体ステップ図

```
[コールドランチ / アプリ起動]
        │
        ▼
[ホーム / カード一覧画面]
        │
        │ タップ: FABカメラアイコン  ← Tap 1
        ▼
[カメラパーミッション確認]  ─── 初回のみ ───► [パーミッション拒否] → [設定誘導ダイアログ]
        │
        │ 許可 (以降はスキップ)
        ▼
[キャプチャ画面 (カメラプレビュー)]
        │
        │ 枠合わせ → シャッタータップ  ← Tap 2 (= カメラ起動後 1タップ)
        ▼
[処理中インジケーター (辺縁検出・透視変換)]  ←  最大3秒
        │
        ├──[検出失敗] ── 全画面エラートースト ──► [再撮影] (キャプチャ画面に戻る)
        │
        ▼
[キャプチャレビュー画面]
        │
        │  フィールド確認・編集 → タグ選択 → イベント選択
        │
        │ タップ: 「保存する」  ← Tap 3
        ▼
[保存処理中インジケーター]
        │
        ├──[保存失敗] ── エラートースト ──► [レビュー画面に残留]
        │
        ▼
[保存完了スナックバー表示]
        │
        ▼
[カード詳細画面] (保存したカードを即表示)
```

---

## タップカウント詳細

| シナリオ | タップ数 | 説明 |
|---|---|---|
| 通常フロー (2回目以降) | 2 | FAB → シャッター (保存はタップ3) |
| コールドランチから撮影 | 2 | ランチャーウィジェット非使用時 |
| ランチャークイックアクション使用 | 1 | OSのショートカットからカメラ直起動 |

カメラ起動 → 撮影準備完了の間に「3タップ」制約に違反するステップは入れない。パーミッションダイアログは OS 管轄のため制約対象外とする。

---

## 各ステップの詳細

### ステップ 1: FABタップ

- トリガー: ホーム / カード一覧のFAB (`photo_camera` アイコン)
- 遷移: キャプチャ画面にフルスクリーンモーダルプッシュ
- アニメーション: FAB展開 → カメラプレビューへの共有要素トランジション (`motion.enter` 300ms)
- パーミッション未取得の場合: キャプチャ画面遷移前に OS パーミッションダイアログを表示

### ステップ 2: カメラプレビューと撮影

- プレビューは フルスクリーン (ステータスバー非表示)
- コーナー検出枠はリアルタイムオーバーレイ
  - 未検出: `color.camera.guide` (白) の 4コーナーガイド
  - 検出済み: `color.camera.guide-active` (ピンク) + ハプティックフィードバック (軽い振動)
- ユーザーはガイドが光ったらシャッターを押す (自動シャッターは MVP対象外)
- フラッシュトグルは左上に常時表示

### ステップ 3: 画像処理 (バックエンド)

- シャッター後、画面は黒フラッシュ → ローディングオーバーレイに切替
- backend が以下を実行:
  1. 四隅座標取得 (OpenCV)
  2. 透視変換・台形補正
  3. 明るさ・コントラスト補正
  4. JPEG/WebP 保存 (≤ 1MB, ≤ 1920px)
  5. OCR (ML Kit) + QR解析 (mobile_scanner)
- タイムアウト: 5秒で中断 → エラーメッセージ表示

### ステップ 4: OCRレビュー画面

- 補正済み画像とOCR結果が並んで表示される
- 信頼度が低いフィールドに `color.semantic.warning` + `!` アイコン
- ユーザーが各フィールドをインライン編集
- タグ・イベントを選択 (どちらも任意)
- 「保存する」ボタンで保存処理

### ステップ 5: 保存と遷移

- 保存後: スナックバー「保存しました！」(`motion.standard` でフェードアウト)
- 遷移先: カード詳細画面 (`replace` でスタックをクリア)
- 「もう1枚追加」のサブアクション: スナックバー内にリンク → キャプチャ画面再表示

## エラー・エッジケース

| 状況 | 表示 | 対処 |
|---|---|---|
| カメラパーミッション拒否 | `capture.permission.denied` ダイアログ | 「設定を開く」で OS設定へ |
| 四隅検出失敗 | トースト「名刺が見つかりませんでした。再度撮影してください」 | キャプチャ画面に留まる |
| OCR結果ゼロ | レビュー画面の全フィールドが空白 + 警告バナー | ユーザーが手動入力して保存可能 |
| 画像が暗すぎる | 警告バナー「画像が暗いです。フラッシュをお試しください」 | 撮影は可能、続行可 |
| ストレージ不足 | `error.storage.full` ダイアログ | 保存をブロック |
| 処理タイムアウト | エラートースト + 「再試行」ボタン | 再撮影またはレビュー画面に戻る |

## ランチャークイックアクション (Phase 7)

iOS ホーム画面の長押しメニュー / Android ショートカットで「名刺をスキャン」を提供。
タップ1回でキャプチャ画面を直接起動 (アプリアイコン → カメラ)。
