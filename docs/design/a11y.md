# アクセシビリティ仕様 — おためいし

Version 1.0

---

## 1. コントラスト比基準

WCAG 2.1 AA 準拠を最低基準とする。

| 用途 | 最低コントラスト比 | 基準 |
|---|---|---|
| 本文テキスト (14sp 以上) | 4.5:1 | WCAG AA |
| 大きなテキスト (18sp 以上 Regular / 14sp 以上 Bold) | 3.0:1 | WCAG AA Large |
| UI コンポーネント枠線・アイコン | 3.0:1 | WCAG AA UI |
| カメラガイド枠 (camera.guide) | 3.0:1 (対暗背景) | — |

### 主要カラーペアのコントラスト確認値

| テキスト色 | 背景色 | モード | 比率 | 判定 |
|---|---|---|---|---|
| `color.text.primary` (#1A1A1A) | `color.surface.primary` (#FFFFFF) | ライト | 18.5:1 | AAA |
| `color.text.primary` (#F0F0F0) | `color.surface.primary` (#121212) | ダーク | 18.1:1 | AAA |
| `color.text.secondary` (#555555) | `color.surface.primary` (#FFFFFF) | ライト | 7.0:1 | AA |
| `color.text.secondary` (#AAAAAA) | `color.surface.primary` (#121212) | ダーク | 5.3:1 | AA |
| `color.text.on-brand` (#FFFFFF) | `color.brand.primary` (#E91E8C) | ライト | 4.6:1 | AA |
| `color.text.on-brand` (#FFFFFF) | `color.brand.primary` (#F06292) | ダーク | 4.5:1 | AA |
| `color.text.link` (#0066CC) | `color.surface.primary` (#FFFFFF) | ライト | 4.6:1 | AA |
| `color.semantic.error` (#C0392B) | `color.surface.primary` (#FFFFFF) | ライト | 5.0:1 | AA |

> `color.text.tertiary` はコントラスト 3.5:1 (ライト) のため、14sp 未満の本文には使用禁止。プレースホルダーおよびタイムスタンプ等の補足的テキストに限定する。

## 2. タッチターゲット

- **最低サイズ**: 44 × 44 dp (iOS HIG / Android Material 推奨)
- ビジュアルサイズが小さいコンポーネント (タグチップ 32dp 高など) は `GestureDetector` の `hitTestBehavior` または `Padding` でタッチ領域を 44dp 以上に拡張すること
- ボトムナビゲーションタブ: 最低 56dp 高 (SystemNavigationBar 除く)
- FAB: 64 × 64dp — 基準を超過し余裕あり
- シャッターボタン: 80 × 80dp — 軍手・汗ばんだ指での操作を想定

### 間隔

隣接するタッチターゲット間の最低間隔: `space.2` (8dp)

## 3. Dynamic Type / Font Scale サポート

| 対応レベル | 内容 |
|---|---|
| iOS | `UIContentSizeCategory` 最大 `accessibilityExtraExtraExtraLarge` まで対応 |
| Android | `fontScale` 最大 2.0 まで対応 |
| Flutter 実装 | `textScaleFactor` を MediaQuery から取得し、コンポーネントがオーバーフローしないこと |

### 具体ルール

1. 固定高さの Row にテキストを入れない。テキスト量に応じて Row → Column へのレスポンシブ切替を検討。
2. カードサムネイルのオーバーレイテキストは最大 2 行で打ち切り、`...` で省略。フォントスケール 200% 時もレイアウト崩壊しないこと。
3. `type.caption` (12sp) は 200% スケールで 24sp 相当になる。このサイズでも読める余白を確保すること。
4. アイコンのみのボタンには必ず `Tooltip` (日本語ラベル) を付けること。

## 4. スクリーンリーダー対応 (VoiceOver / TalkBack)

### 基本方針

- すべてのインタラクティブ要素に `Semantics` ウィジェットで日本語ラベルを付与する
- 画像カードには画像説明 (`semanticsLabel`) を設定する (内容不明時は「名刺画像」)
- 装飾的な画像・アイコンは `excludeFromSemantics: true` を指定する

### セマンティックラベル命名規則

```
[要素の種類] [固有情報] [状態] [操作ヒント]
```

例:
- `"名刺、推しさん、2025年8月17日保存、ダブルタップで詳細を開く"`
- `"タグ、Vtuber、選択済み、ダブルタップで選択解除"`
- `"シャッターボタン、ダブルタップで撮影"`
- `"イベント、コミケ106、2024年12月保存済み、ダブルタップで詳細を開く"`

### フォーカス順序

各画面のフォーカス順序は論理的な読み上げ順 (上→下、左→右) とする。

モーダル・ボトムシート表示時はフォーカスをモーダル内にトラップし、閉じた後に直前のフォーカス位置に戻すこと。

### アナウンス

- 保存完了: `"名刺を保存しました"` をスクリーンリーダーに通知 (`AccessibilityAnnouncement`)
- エラー: エラーメッセージを即時アナウンス
- OCR完了: `"テキストを読み取りました。確認してください"` をアナウンス

## 5. カラーだけに依存しない状態表現

| 状態 | 色 | 追加の視覚手がかり |
|---|---|---|
| タグ選択済み | `color.brand.secondary` | チェックアイコン表示 |
| エラー入力フィールド | `color.semantic.error` 枠線 | エラーアイコン + エラーテキスト |
| OCR要確認 | `color.semantic.warning` | `!` アイコン + ラベルテキスト |
| 読み込み中 | — | スピナー + `"読み込み中"` テキスト |

## 6. モーション・アニメーション

- `prefers-reduced-motion` 相当の設定 (iOS: 視差効果を減らす / Android: アニメーション無効化) が ON の場合、`motion.standard` 以上の duration を 0ms にすること
- スクロールアニメーション・パララックスは実装しない
- カメラプレビュー内のガイド枠アニメーションは reduced motion でも維持する (機能的フィードバックのため)

---

## 7. キーボード・外部入力

- 外付けキーボード接続時にすべてのフォームが操作可能であること
- Tab キーでフォーカス移動、Enter / Space でアクション実行
- カメラシャッターはスペースキーまたはボリュームボタンで代替可能にすること (実装は backend 担当だが設計上の要件として明記)

---

## 8. エラーと支援テキスト

- フォーム検証エラーは該当フィールドの直下に表示 (ダイアログで代替しない)
- エラーテキストは `color.semantic.error` + エラーアイコンで提示
- 支援テキスト (ヒント) は `color.text.tertiary` で入力フィールド下に常時表示

---

## 9. ロケールと文字

- アプリ言語: 日本語 (ja-JP) を第一言語として設定
- 英語 (en-US) は l10n パッケージ経由でのみ提供
- 日本語テキストには常に `locale: Locale('ja', 'JP')` を指定し、文字レンダリングを正確にすること
