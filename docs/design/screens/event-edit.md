# イベント追加・編集画面

Version 1.0

---

## 1. 目的

新規イベントの作成 (`+` FAB から)、および既存イベントの編集 (詳細画面 → 編集)。同じ画面が新規/編集の両方をハンドルする。

---

## 2. エントリポイント

- イベント一覧 FAB → 新規作成 (modal push, route `/event/edit`)
- イベント詳細画面 「編集」アクション → 既存編集 (route `/event/edit`, arguments: eventId)
- 確認画面 (capture-review) のイベント選択 → 「+ 新規イベントを作成」→ 同上

---

## 3. レイアウト (ASCII)

```
┌─────────────────────────────────┐
│ ✕  イベントを追加 / 編集         │  ← AppBar: 左 close, 右 アクションなし
├─────────────────────────────────┤
│                                 │
│ イベント名 *                    │
│ ┌─────────────────────────────┐ │
│ │ コミケ106                    │ │  ← TextField, maxLength 60
│ └─────────────────────────────┘ │
│                                 │
│ 開始日                          │
│ ┌─────────────────────────────┐ │
│ │ 2024年12月29日       [📅]    │ │  ← 日付 chip-style, タップで DatePicker
│ └─────────────────────────────┘ │
│                                 │
│ 終了日 (任意)                   │
│ ┌─────────────────────────────┐ │
│ │ 2024年12月31日       [📅]    │ │  ← 開始日と同じ
│ └─────────────────────────────┘ │
│                                 │
│ 会場 (任意)                     │
│ ┌─────────────────────────────┐ │
│ │ 東京ビッグサイト              │ │  ← TextField, maxLength 80
│ └─────────────────────────────┘ │
│                                 │
│ メモ (任意)                     │
│ ┌─────────────────────────────┐ │
│ │ (3行 multiline)               │ │
│ └─────────────────────────────┘ │
│                                 │
│ [ 保存                       ]  │  ← ElevatedButton (フル幅)
│ [ 削除 (編集時のみ)            ]│  ← OutlinedButton, color: error
└─────────────────────────────────┘
```

---

## 4. フィールド

| フィールド | 必須 | 型 | バリデーション |
|---|---|---|---|
| 名前 | ✓ | String | 1〜60 文字、trim 後の空白のみ不可 |
| 開始日 | – | DateTime? | 任意、未設定で保存可 |
| 終了日 | – | DateTime? | 任意、設定時は **開始日 ≥ 終了日不可** (エラー表示) |
| 会場 | – | String? | 0〜80 文字 |
| メモ | – | String? | 0〜500 文字、改行可 |

開始日 = null かつ 終了日 = null は許容 (「日付未設定」イベント)。
開始日 = null かつ 終了日 = not null は許容しない (UI で保存ブロック)。

---

## 5. アクション

| ボタン | 表示条件 | 挙動 |
|---|---|---|
| 保存 | 常時 | `EventRepository.insert / update` → 前画面に戻る |
| 削除 | 編集モードのみ | 確認ダイアログ → `EventRepository.delete` → 一覧に戻る |
| 閉じる ✕ | 常時 | 未保存変更ありなら確認ダイアログ |

---

## 6. 確認ダイアログ

### 削除確認
```
タイトル: イベントを削除しますか?
本文:    紐づくカードは残りますが、イベント情報は失われます。
削除:    削除 (error color)
キャンセル: キャンセル
```

### 未保存変更の破棄
```
タイトル: 編集中の内容を破棄しますか?
削除:    破棄
キャンセル: 編集を続ける
```

---

## 7. 実装メモ

- ファイル: NEW `lib/screens/event_edit_screen.dart`
- ルート: `'/event/edit'`、`onGenerateRoute` で `arguments: String? eventId` 受け取り
- 既存 `EventDetailScreen` から push する場合は `eventId` 引数あり
- DatePicker は `showDatePicker(initialDate, firstDate: 1990, lastDate: 2099, locale: const Locale('ja'))`
- 保存後は前画面に `pop(true)` を返し、呼び出し側で provider invalidate
- 新規 vs 編集の AppBar タイトル切替: 引数 `eventId == null ? 'イベントを追加' : 'イベントを編集'`
