# キャプチャレビュー画面 v3 — タグチップ + イベント選択

Version 3.0 — extends `capture-review-v2.md`. v2 (編集可能 X / URL リスト + manual モード) は維持しつつ、以下を追加:

1. タグ入力をプレーンテキストから **既存タグの multiselect chips + 新規入力** に置き換え
2. **イベント選択** UI を追加 (既存イベント + 新規作成)

---

## 1. タグ UI

### 1-1. レイアウト (タグ欄のみ抜粋)

```
タグ
┌─────────────────────────────────┐
│ [Vtuber ×][コス ×][絵描き ×]    │  ← 選択済みチップ
│                                 │
│ [+ タグを追加 ▼]                │  ← ボタン (タップでボトムシート)
└─────────────────────────────────┘
```

### 1-2. タグ追加ボトムシート

```
┌─────────────────────────────────┐
│ ─                                │
│ タグを選択                       │
├─────────────────────────────────┤
│ 🔍 タグを検索 or 新規作成         │  ← TextField
├─────────────────────────────────┤
│ ☑ Vtuber                          │
│ ☐ コス                            │
│ ☐ 絵描き                          │
│ ☐ ホロライブ                       │
│ ...                              │
├─────────────────────────────────┤
│ [+ 新しいタグ「○○○」を作成]      │  ← 検索入力に一致なしの時表示
│                                  │
│ [選択をクリア]  [完了 (3)]         │
└─────────────────────────────────┘
```

- 既存タグ: `findAll()` をボトムシート open 時にロード、ABC / あいうえお順
- 検索入力: 部分一致でリストをフィルタ。完全一致するタグがなければ「+ 新しいタグ "{query}" を作成」を追加可能項目として表示
- 完了タップで親に選択タグ ID 配列 + 新規作成名配列を返す

### 1-3. 保存時の確認 (新規タグ作成あり)

新規タグが含まれる場合、保存ボタン押下時に確認ダイアログ:

```
タイトル: 新しいタグを作成しますか?
本文:    以下のタグは初めて使用されます:
         • 推し活
         • 新刊
作成して保存: 作成して保存
キャンセル:   キャンセル
```

OK → `TagRepository.findOrCreate` で作成 → `SaveCardUseCase.execute` で関連付け。

### 2-1. レイアウト (イベント欄、X アカウントとリンクの間に挿入)

```
イベント (任意)
┌─────────────────────────────────┐
│  コミケ106  (2024年12月29日)  [×]│  ← 選択時: chip 風表示
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ [+ イベントを選択 ▼]             │  ← 未選択時
└─────────────────────────────────┘
```

### 2-2. イベント選択ボトムシート

```
┌─────────────────────────────────┐
│ ─                                │
│ イベントを選択                   │
├─────────────────────────────────┤
│ 🔍 イベント名を検索               │
├─────────────────────────────────┤
│ ○ コミケ106  2024年12月29日       │
│ ● にじフェス2026  2026年3月15日   │  ← 選択中 (radio)
│ ○ 蓮ノ空 4th LIVE  2026年8月23日  │
│ ○ オフ会  日付未設定              │
├─────────────────────────────────┤
│ [+ 新しいイベントを作成]          │  ← タップで `/event/edit` に push
│ [選択を解除]  [完了]              │
└─────────────────────────────────┘
```

- イベントは 1 件のみ選択可 (`RadioListTile`)
- 「新しいイベントを作成」タップ → 現在のシートを閉じ、`/event/edit` (新規モード) に push。保存後にイベント ID を pop 結果で受け取り、選択状態に反映
- 「選択を解除」で eventId = null

## 3. 保存ロジックの変更

`SaveCardInput` には既に `eventId: String?` と `tagNames: List<String>` のフィールドあり。タグについて以下のように変更:

```dart
// 既存:
tagNames: _tags.text.split(...).where(...).toList(),

// 新:
tagNames: _resolvedTags.map((t) => t.name).toList()
       + _pendingNewTags,  // 新規作成予約分
```

`SaveCardUseCase.execute` は既に `tagNames` を `findOrCreate` で処理しているので、新規タグ名を渡せば自動作成される。確認ダイアログは UI 側で出すだけ。

---

## 4. 状態管理

```dart
class _ReviewState {
  final List<Tag> selectedExistingTags;   // 既存タグの選択
  final List<String> pendingNewTagNames;  // 確認後に作成する新規タグ名
  final Event? selectedEvent;             // 選択中イベント (null = 未選択)
}
```

タグ・イベント情報は `ref.read(tagRepositoryProvider.future)` / `ref.read(eventRepositoryProvider.future)` で取得しキャッシュ。ボトムシート open 時にロード。
