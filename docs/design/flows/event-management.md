# イベント管理フロー

Version 1.0

---

## フロー概要

イベントの作成・編集・削除、および名刺へのイベント紐付け管理。

---

## 全体ステップ図

```
[イベント一覧 (ボトムナビ「イベント」)]
        │
        ├── 既存イベントタップ ──────────────► [イベント詳細]
        │                                             │
        │                                    [カードグリッド閲覧]
        │                                    [編集ボタン] → [イベント編集フォーム]
        │                                    [削除] → 確認ダイアログ → 一覧に戻る
        │
        └── 「イベントを追加」FABタップ ───► [イベント作成フォーム]
                                                      │
                                               「保存する」 → [イベント詳細]
```

### 名刺保存時のイベント紐付けフロー

```
[キャプチャレビュー画面]
        │
        「イベントを選択」タップ
        ▼
[イベント選択ボトムシート]
  ├── 既存イベントリスト (日付降順)
  ├── 「イベントなし」選択肢 (デフォルト)
  └── 「新しいイベントを作成」 → [イベント作成フォーム (モーダル)]
                                        │
                                 保存 → ボトムシートに戻り、新イベントが選択状態
```

---

## イベント作成フォーム仕様

### フィールド

| フィールド | 型 | 必須 | バリデーション |
|---|---|---|---|
| イベント名 | テキスト | 必須 | 最大50文字、空白不可 |
| 開催日 | 日付ピッカー | 任意 | 過去・未来どちらも可 |
| メモ | テキストエリア | 任意 | 最大500文字 |

### 保存ボタン

- イベント名が空白の場合: 保存ボタンを非活性 (グレー)
- 保存中: ボタン内スピナー
- 保存完了: フォームを閉じてイベント詳細 or 呼び出し元に戻る

---

## イベント削除の挙動

1. 削除確認ダイアログ表示 (`event-detail.delete.confirm`)
2. 確認後: イベントレコードを削除
3. 紐付き名刺の `event_id` を `NULL` にクリア (名刺は削除しない)
4. イベント一覧に戻る

---

## エラー・エッジケース

| 状況 | 表示 |
|---|---|
| イベント0件 | `event-list.empty` 空状態 |
| 同名イベント作成 | 許可 (ユーザーが意図的に複数回参加の場合があるため) |
| イベント名が50文字超 | インラインバリデーションエラー |
| 開催日なし | `event-detail.date.unknown` を表示 |
| イベント削除後のカード | 詳細画面でイベント欄が `detail.no-event` になる |
